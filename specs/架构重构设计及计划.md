## 执行摘要

本重构计划的核心目标是构建**瘦 Extension 架构**：

| 职责 | Extension | Main App |
|------|-----------|----------|
| 监听范围 | ✅ 全盘（/Users/ + 外接磁盘） | - |
| 数据读取 | ❌ 不读取 | ✅ SwiftData |
| 菜单构建 | ❌ 只渲染 | ✅ 构建配置 |
| 业务逻辑 | ❌ 无 | ✅ 全部 |
| 完全磁盘访问权限 | ❌ 不处理 | ✅ 引导用户授权 |
| 图标加载 | ✅ 根据路径 | ✅ 预加载缓存 |

**核心原则**：Extension 只做 3 件事——监听文件目录、渲染菜单、发送事件。

### 预期收益

- Extension 内存使用 < 20MB（无 SwiftData 依赖）
- 菜单渲染时间 < 50ms
- 代码行数减少约 30%
- 维护成本降低约 40%
- 用户体验提升：一次授权全盘访问，无需手动配置许可目录
- 新增脚本执行功能：支持 Shell/Python/AppleScript 三种脚本类型

---

## 设计原则

1. **单一真相源**：主 App 保存 MenuSnapshot 作为状态真相源
2. **推拉混合模式**：主 App 推送配置，Extension 可主动请求
3. **版本检查**：防止重复和乱序消息
4. **幂等处理**：重复消息不产生副作用
5. **完全磁盘访问权限**：用户手动授权，无需 Bookmark 缓存

## 关键文件

### 需要创建

- `RClick/Shared/CommunicationProtocol.swift` - 类型安全的通信协议
- `RClick/Shared/IconCache.swift` - 统一 Icon 缓存
- `RClick/Model/ScriptEntity.swift` - 脚本 SwiftData 实体
- `RClick/Settings/ScriptsSettingsTabView.swift` - 脚本设置界面

- `FinderSyncExt/FinderSyncExt.swift` - 精简为纯渲染层
- `RClick/Shared/Messager.swift` - 支持类型安全消息

- `RClick/RClickApp.swift` - 处理点击事件和脚本执行
