## 零、执行摘要

本重构计划的核心目标是构建**瘦 Extension 架构**：

| 职责 | Extension | Main App |
|------|-----------|----------|
| 监听范围 | ✅ 全盘（/Users/ + 外接磁盘） | - |
| 数据读取 | ❌ 不读取 | ✅ SwiftData |
| 菜单构建 | ❌ 只渲染 | ✅ 构建配置 |
| 业务逻辑 | ❌ 无 | ✅ 全部 |
| 权限请求 | ❌ 不处理 | ✅ 按需请求 |
| Bookmark 缓存 | ❌ 不处理 | ✅ 缓存授权文件夹 |
| 图标加载 | ✅ 根据路径 | ✅ 预加载缓存 |

**核心原则**：Extension 只做 3 件事——监听文件目录、渲染菜单、发送事件。

## 一、当前问题总结

### 1.1 架构层面
- **重复实现**: Extension 有 `IconCache`，Main App 有 `IconCacheManager`
- **数据模型冗余**: RCAction/ActionMenuItem、OpenWithApp/AppMenuItem 需要转换
- **通信协议简陋**: `MessagePayload.action` 使用 String 类型，缺乏类型安全
- **转换层复杂**: 需要在 SwiftData 模型和 MenuItem 模型间转换

### 1.2 性能问题
- **频繁 JSON 编解码**: 每次菜单更新都需要通过 UserDefaults 编解码
- **Tag 管理低效**: 使用随机数生成 tag，可能冲突导致性能问题
- **无缓存机制**: 图标每次都重新加载

### 1.3 数据一致性
- **同步延迟**: 配置更新需要等待通知和 UserDefaults 刷新
- **转换错误风险**: 模型转换可能丢失数据或引入 bug

### 1.4 用户体验
- **Main App 依赖**: Extension 在 Main App 关闭时无法工作
- **更新延迟**: 配置更新实时性差（依赖通知 + 数据转换）

## 二、设计目标

### 2.1 核心原则
1. **简洁性**: 减少重复代码，统一 Icon 缓存实现
2. **类型安全**: 使用枚举替代字符串，编译时检查
3. **性能**: 优化 Tag 管理，使用递增 ID；图标预加载缓存
4. **可靠性**: 错误处理完善，日志记录清晰

### 2.2 架构选择：瘦 Extension

Extension 只做 3 件事：
1. **接收消息** - 从主程序接收菜单配置、刷新指令
2. **渲染菜单** - 根据收到的配置渲染菜单（不读任何数据源）
3. **发送事件** - 用户点击后发送操作数据给主程序

**所有业务逻辑在主程序完成。**

### 2.3 核心改进（按优先级）

| 优先级 | 改进项 | 预计时间 | 风险 |
|--------|--------|----------|------|
| P0 | 类型安全通信协议 | 1-2 小时 | 低 |
| P0 | 瘦 Extension 架构 | 2-3 小时 | 中 |
| P1 | 优化 Tag 管理 | 30 分钟 | 极低 |
| P1 | 统一 Icon 缓存 | 1 小时 | 低 |
| P2 | 文档翻译/注释 | 1 小时 | 无 |

### 3.2 新架构职责划分

#### Main App 职责
- **数据管理**: 读取 SwiftData，构建菜单配置
- **业务逻辑**: 处理所有动作执行（复制路径、打开应用等）
- **配置推送**: 数据变更时发送完整菜单配置给 Extension
- **事件处理**: 接收 Extension 的点击事件，执行对应操作
- **权限请求**: 按需请求文件访问权限（NSOpenPanel）
- **Bookmark 缓存**: 缓存用户授权过的文件夹 bookmark 数据

#### Extension 职责
- **文件监听**: 监听全盘文件变化（默认 `/Users/`，自动检测外接磁盘）
- **消息监听**: 监听来自主程序的配置消息
- **菜单渲染**: 根据配置渲染 NSMenuItem
- **事件转发**: 用户点击后发送事件给主程序
- **心跳发送**: 定期发送心跳确认存活

### 3.3 Extension 监听范围

```swift
// FinderSyncExt.swift
override init() {
    super.init()

    // 默认监听 /Users/
    let usersDir = URL(fileURLWithPath: "/Users/")

    // 检测并添加外接磁盘
    let volumeURLs = FileManager.default.mountedVolumeURLs(
        includingResourceValuesForKeys: [.isBrowsableKey],
        options: [.skipHiddenVolumes]
    ) ?? []

    var allURLs = [usersDir]
    for volume in volumeURLs {
        if volume.path != "/" {  // 排除根目录
            allURLs.append(volume)
        }
    }

    FIFinderSyncController.default().directoryURLs = Set(allURLs)

    logger.info("Monitoring directories: \(allURLs.map { $0.path })")

    setupMessageHandlers()
    sendHeartbeat()
}
```

#### 监听策略
- **默认监听**: `/Users/`（用户主目录）
- **外接磁盘**: 自动检测并添加所有可浏览的挂载卷
- **网络卷**: 不主动添加（由用户授权后按需访问）

### 4.4 Messager 类设计（Notification + 协议层封装）

基于 `DistributedNotificationCenter` + Codable 协议层封装：

```swift
import Foundation

// MARK: - 类型安全的消息协议

/// 主程序 → Extension 消息类型
enum MainToExtensionAction: String, Codable {
    case menuConfig = "menu-config"
    case running = "running"
    case quit = "quit"
    case requestConfig = "request-config"
}

/// Extension → 主程序消息类型
enum ExtensionToMainAction: String, Codable {
    case click = "click"
    case heartbeat = "heartbeat"
}

// MARK: - 消息载体（带协议层封装）

/// 主程序发送给 Extension 的消息
struct MainToExtensionMessage: Codable {
    let id: UUID              // 消息 ID，用于追踪
    let action: MainToExtensionAction
    let data: Data?           // JSON 编码的载荷

    init(id: UUID = UUID(),
         action: MainToExtensionAction,
         data: Codable? = nil) {
        self.id = id
        self.action = action
        self.data = try? JSONEncoder().encode(data)
    }
}

/// Extension 发送给主程序的消息
struct ExtensionToMainMessage: Codable {
    let id: UUID              // 消息 ID，用于追踪
    let action: ExtensionToMainAction
    let data: Data?           // JSON 编码的载荷

    init(id: UUID = UUID(),
         action: ExtensionToMainAction,
         data: Codable? = nil) {
        self.id = id
        self.action = action
        self.data = try? JSONEncoder().encode(data)
    }
}

// MARK: - Messager 类

class Messager {
    static let shared = Messager()

    private let center: DistributedNotificationCenter = .default()

    // 消息处理器存储
    private var mainToExtensionHandlers: [MainToExtensionAction: (Data?) -> Void] = [:]
    private var extensionToMainHandlers: [ExtensionToMainAction: (Data?) -> Void] = [:]

    // 通知名称
    static let mainToExtensionNotification = "RClick.MainToExtension"
    static let extensionToMainNotification = "RClick.ExtensionToMain"

    private init() {
        // 注册通知监听
        center.addObserver(
            self,
            selector: #selector(handleMainToExtension(_:)),
            name: NSNotification.Name(Self.mainToExtensionNotification),
            object: nil
        )

        center.addObserver(
            self,
            selector: #selector(handleExtensionToMain(_:)),
            name: NSNotification.Name(Self.extensionToMainNotification),
            object: nil
        )
    }

    // MARK: - 发送消息

    /// 主程序发送消息给 Extension
    func sendToExtension(_ action: MainToExtensionAction, data: Codable? = nil) {
        let message = MainToExtensionMessage(action: action, data: data)
        sendMessage(message, via: Self.mainToExtensionNotification)
    }

    /// Extension 发送消息给主程序
    func sendToMain(_ action: ExtensionToMainAction, data: Codable? = nil) {
        let message = ExtensionToMainMessage(action: action, data: data)
        sendMessage(message, via: Self.extensionToMainNotification)
    }

    private func sendMessage<T: Codable>(_ message: T, via notification: String) {
        guard let messageData = try? JSONEncoder().encode(message) else { return }
        let messageString = String(data: messageData, encoding: .utf8) ?? ""

        center.postNotificationName(
            NSNotification.Name(notification),
            object: messageString,
            userInfo: nil,
            deliverImmediately: true
        )
    }

    // MARK: - 注册处理器

    /// Extension 注册主程序消息处理器
    func onMainMessage(_ action: MainToExtensionAction, handler: @escaping (Data?) -> Void) {
        mainToExtensionHandlers[action] = handler
    }

    /// 主程序注册 Extension 消息处理器
    func onExtensionMessage(_ action: ExtensionToMainAction, handler: @escaping (Data?) -> Void) {
        extensionToMainHandlers[action] = handler
    }

    // MARK: - 便捷方法：解码数据

    func decode<T: Codable>(_ data: Data?) -> T? {
        guard let data = data else { return nil }
        return try? JSONDecoder().decode(T.self, from: data)
    }

    // MARK: - 通知处理

    @objc private func handleMainToExtension(_ notification: NSNotification) {
        guard let messageString = notification.object as? String,
              let messageData = messageString.data(using: .utf8),
              let message = try? JSONDecoder().decode(MainToExtensionMessage.self, from: messageData) else {
            return
        }

        mainToExtensionHandlers[message.action]?(message.data)
    }

    @objc private func handleExtensionToMain(_ notification: NSNotification) {
        guard let messageString = notification.object as? String,
              let messageData = messageString.data(using: .utf8),
              let message = try? JSONDecoder().decode(ExtensionToMainMessage.self, from: messageData) else {
            return
        }

        extensionToMainHandlers[message.action]?(message.data)
    }
}

// MARK: - 便捷扩展

extension Messager {
    /// 主程序发送菜单配置给 Extension
    func sendMenuConfig(_ config: MenuConfigPayload) {
        sendToExtension(.menuConfig, data: config)
    }

    /// Extension 发送点击事件给主程序
    func sendClickEvent(_ event: ClickEventPayload) {
        sendToMain(.click, data: event)
    }
}
```

### 5.1 统一 Icon 缓存管理

#### 问题
- Extension 中有 `IconCache` 类
- Main App 中有 `IconCacheManager` 类
- 两者功能相同但未共享

#### 解决方案
将 Icon 缓存移至共享模块，在 `RClick/Shared/` 中创建统一的实现：

**新建文件**: `RClick/Shared/IconCache.swift`

```swift
@MainActor
public class IconCache {
    public static let shared = IconCache()

    private var memoryCache: [String: NSImage] = [:]
    private let iconSize = CGSize(width: 32, height: 32)

    private init() {}

    /// 获取 URL 对应的图标（带缓存）
    public func icon(for url: URL) -> NSImage {
        let cacheKey = url.path
        if let cached = memoryCache[cacheKey] {
            return cached
        }

        let icon = NSWorkspace.shared.icon(forFile: url.path)
        icon.size = iconSize
        memoryCache[cacheKey] = icon
        return icon
    }

    /// 批量预加载图标
    public func preloadIcons(for urls: [URL]) {
        for url in urls {
            _ = icon(for: url)
        }
    }

    /// 清理缓存
    public func clearCache() {
        memoryCache.removeAll()
    }
}
```

**修改文件**:
- 删除 `FinderSyncExt/FinderSyncExt.swift` 中的 IconCache 类
- 简化 `RClick/Shared/Messager.swift` 中的 IconCacheManager，直接使用 IconCache

### 5.3 优化 Tag 管理

#### 问题
使用随机数生成 tag 可能冲突：
```swift
private func getUniqueTag(for rid: String) -> Int {
    var newTag = Int.random(in: 1 ... Int.max)
    while tagRidDict.keys.contains(newTag) {  // 可能循环很久
        newTag = Int.random(in: 1 ... Int.max)
    }
    tagRidDict[newTag] = rid
    return newTag
}
```

#### 解决方案：使用递增 ID

```swift
@MainActor
class FinderSyncExt: FIFinderSync {
    private var nextTag: Int = 1  // 递增计数器

    private func getUniqueTag(for rid: String) -> Int {
        let tag = nextTag
        nextTag += 1
        tagRidDict[tag] = rid
        return tag
    }
}
```

#### 优点
- ✅ 避免随机数冲突
- ✅ 性能更好（O(1) vs 可能的 O(n)）
- ✅ 行为可预测

### 5.4 Extension 精简版代码

```swift
// FinderSyncExt.swift - 精简版
import Cocoa
import FinderSync

@MainActor
class FinderSyncExt: FIFinderSync {
    private var menuConfig: MenuConfigPayload?
    private var nextTag: Int = 1
    private var tagItemDict: [Int: (id: String, type: MenuItemType)] = [:]
    private var isHostAppOpen = false
    private var currentMenuKind: FIMenuKind = .contextualMenuForContainer

    override init() {
        super.init()
        FIFinderSyncController.default().directoryURLs = [URL(fileURLWithPath: "/Users/")]
        setupMessageHandlers()
        sendHeartbeat()
    }

    private func setupMessageHandlers() {
        Messager.shared.onMainMessage(.menuConfig) { [weak self] data in
            self?.menuConfig = Messager.shared.decode(data)
            if let dirs = self?.menuConfig?.observeDirs {
                FIFinderSyncController.default().directoryURLs = Set(dirs.map { URL(fileURLWithPath: $0) })
            }
        }

        Messager.shared.onMainMessage(.running) { [weak self] _ in
            self?.isHostAppOpen = true
            self?.requestMenuConfig()
        }

        Messager.shared.onMainMessage(.quit) { [weak self] _ in
            self?.isHostAppOpen = false
        }
    }

    override func menu(for menuKind: FIMenuKind) -> NSMenu {
        currentMenuKind = menuKind
        guard isHostAppOpen, let config = menuConfig else {
            return NSMenu(title: "RClick")
        }

        let menu = NSMenu(title: "RClick")

        // App menu items
        for app in config.apps {
            menu.addItem(createAppMenuItem(app))
        }

        // New file items
        if let newFileItem = createNewFileSubmenu(config.newFiles) {
            menu.addItem(newFileItem)
        }

        // Common dir items
        if let commonDirItem = createCommonDirSubmenu(config.commonDirs) {
            menu.addItem(commonDirItem)
        }

        // Separator
        menu.addItem(NSMenuItem.separator())

        // Action items
        for action in config.actions where action.enabled {
            menu.addItem(createActionMenuItem(action))
        }

        return menu
    }

    private func createAppMenuItem(_ app: AppMenuItem) -> NSMenuItem {
        let item = NSMenuItem(title: app.name, action: #selector(handleMenuClick(_:)), keyEquivalent: "")
        item.tag = nextTag
        tagItemDict[nextTag] = (app.id, .app)
        nextTag += 1

        // 根据应用路径加载图标
        let icon = NSWorkspace.shared.icon(forFile: app.appURL.path)
        icon.size = NSSize(width: 16, height: 16)
        item.image = icon
        return item
    }

    private func createActionMenuItem(_ action: ActionMenuItem) -> NSMenuItem {
        let item = NSMenuItem(title: action.name, action: #selector(handleMenuClick(_:)), keyEquivalent: "")
        item.tag = nextTag
        tagItemDict[nextTag] = (action.id, .action)
        nextTag += 1

        if let icon = NSImage(systemSymbolName: action.icon, accessibilityDescription: action.name) {
            icon.size = NSSize(width: 16, height: 16)
            item.image = icon
        }
        item.isEnabled = action.enabled
        return item
    }

    @objc private func handleMenuClick(_ sender: NSMenuItem) {
        guard let (itemId, itemType) = tagItemDict[sender.tag] else { return }

        let event = ClickEventPayload(
            action: .click,
            itemId: itemId,
            itemType: itemType,
            target: getTargetPaths(),
            trigger: getTriggerKind(),
            itemTypes: getItemTypes()
        )

        Messager.shared.sendClickEvent(event)
    }

    private func sendHeartbeat() {
        Messager.shared.sendToMain(.heartbeat)
    }

    private func getTargetPaths() -> [String] {
        switch currentMenuKind {
        case .contextualMenuForItems:
            return FIFinderSyncController.default().selectedItemURLs()?.map { $0.path() } ?? []
        default:
            return [FIFinderSyncController.default().targetedURL()?.path()].compactMap { $0 }
        }
    }

    private func getItemTypes() -> [ItemType] {
        let fm = FileManager.default
        return getTargetPaths().map { path in
            var isDir: ObjCBool = false
            fm.fileExists(atPath: path, isDirectory: &isDir)
            return isDir.boolValue ? .folder : .file
        }
    }

    private func getTriggerKind() -> MenuTrigger {
        switch currentMenuKind {
        case .contextualMenuForItems: return .contextualItems
        case .contextualMenuForContainer: return .contextualContainer
        case .contextualMenuForSidebar: return .contextualSidebar
        case .toolbarItemMenu: return .toolbar
        default: return .unknown
        }
    }

    private func requestMenuConfig() {
        Messager.shared.sendToMain(.requestConfig)
    }
}
```

### 5.3 Bookmark 缓存管理

```swift
// RClick/Shared/BookmarkManager.swift
import Foundation

/// Bookmark 管理器 - 缓存用户授权的文件夹访问权限
actor BookmarkManager {
    static let shared = BookmarkManager()

    private var bookmarks: [String: Data] = [:]  // path -> bookmark data
    private let userDefaultsKey = "FolderBookmarks"

    private init() {
        loadBookmarks()
    }

    /// 加载保存的 bookmarks
    private func loadBookmarks() {
        if let data = UserDefaults.standard.data(forKey: userDefaultsKey),
           let decoded = try? JSONDecoder().decode([String: Data].self, from: data) {
            bookmarks = decoded
        }
    }

    /// 保存 bookmarks 到持久化存储
    private func saveBookmarks() {
        if let encoded = try? JSONEncoder().encode(bookmarks) {
            UserDefaults.standard.set(encoded, forKey: userDefaultsKey)
        }
    }

    /// 为 URL 创建 bookmark
    func createBookmark(for url: URL) throws -> Data {
        let bookmark = try url.bookmarkData(
            options: [.withSecurityScope],
            includingResourceValuesForKeys: nil,
            relativeTo: nil
        )
        bookmarks[url.path] = bookmark
        saveBookmarks()
        return bookmark
    }

    /// 从 bookmark 恢复 URL 访问
    func restoreAccess(from bookmark: Data, for path: String) -> URL? {
        var isStale = false
        guard let url = try? URL(
            resolvingBookmarkData: bookmark,
            options: [.withSecurityScope],
            relativeTo: nil,
            bookmarkDataIsStale: &isStale
        ) else {
            return nil
        }

        // 如果 bookmark 已过期，尝试更新
        if isStale, let freshBookmark = try? createBookmark(for: url) {
            bookmarks[path] = freshBookmark
            saveBookmarks()
        }

        return url
    }

    /// 获取 URL 的 bookmark（如果存在）
    func getBookmark(for path: String) -> Data? {
        return bookmarks[path]
    }

    /// 删除 bookmark
    func removeBookmark(for path: String) {
        bookmarks.removeValue(forKey: path)
        saveBookmarks()
    }

    /// 访问授权的文件（自动管理 security scope）
    func accessFile<T>(at path: String, operation: (URL) throws -> T) -> T? {
        guard let bookmark = bookmarks[path],
              let url = restoreAccess(from: bookmark, for: path) else {
            return nil
        }

        guard url.startAccessingSecurityScopedResource() else {
            return nil
        }

        defer {
            url.stopAccessingSecurityScopedResource()
        }

        return try? operation(url)
    }
}
```

### 5.4 权限处理流程

```swift
// RClick/RClickApp.swift
extension RClickApp {

    /// 处理需要权限的文件操作
    func handleActionWithPermission(
        itemId: String,
        target: [String],
        trigger: String,
        operation: @escaping ([URL]) -> Void
    ) {
        let urls = target.map { URL(fileURLWithPath: $0) }

        // 检查是否有 bookmark 缓存
        let bookmarkedURLs = urls.compactMap { path -> URL? in
            Task {
                await BookmarkManager.shared.getBookmark(for: path).flatMap {
                    BookmarkManager.shared.restoreAccess(from: $0, for: path)
                }
            }
        }

        // 如果有未授权的 URL，请求权限
        let unauthorizedURLs = urls.filter { url in
            !bookmarkedURLs.contains(where: { $0.path == url.path })
        }

        guard unauthorizedURLs.isEmpty else {
            // 需要请求权限
            requestAccess(for: unauthorizedURLs) { grantedURLs in
                // 保存新的 bookmarks
                Task {
                    for url in grantedURLs {
                        try? await BookmarkManager.shared.createBookmark(for: url)
                    }
                }
                // 执行操作
                operation(urls)
            }
            return
        }

        // 所有文件都已授权，直接执行
        operation(urls)
    }

    /// 请求文件访问权限
    private func requestAccess(
        for urls: [URL],
        completion: @escaping ([URL]) -> Void
    ) {
        let panel = NSOpenPanel()
        panel.canChooseFiles = true
        panel.canChooseDirectories = true
        panel.allowsMultipleSelection = true
        panel.directoryURLs = urls
        panel.begin { response in
            if response == .OK {
                completion(panel.urls)
            } else {
                completion([])
            }
        }
    }
}
```

## 九、构建和发布重构方案

### 9.1 现状与问题

当前 RClick 项目使用传统的 Xcode 工程模式，存在以下问题：

| 问题 | 描述 | 影响 |
|------|------|------|
| 工程配置不透明 | `.xcodeproj` 是二进制状态文件，难以 diff 和合并 | Git 冲突频繁 |
| 文件需手动拖拽 | 新文件必须在 Xcode 中手动添加到 target | AI 无法自动完成 |
| 依赖管理不可复现 | 依赖通过 UI 添加，配置写死在 pbxproj | CI 配置困难 |
| 构建操作不统一 | 依赖点击 Xcode 按钮 | 无法自动化 |
| 发布流程繁琐 | 手动签名、公证、打包、上传 | 每次 1-2 小时 |

```yaml
# .github/workflows/release.yml
name: Build, Sign, Notarize and Release

on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write

jobs:
  build-and-release:
    runs-on: macos-14

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Get Version
        id: version
        run: |
          VERSION=${GITHUB_REF#refs/tags/v}
          echo "version=$VERSION" >> $GITHUB_OUTPUT

      # 1. 导入签名证书
      - name: Import Certificate
        env:
          CERTIFICATE_P12: ${{ secrets.MACOS_CERT_P12 }}
          CERTIFICATE_PASSWORD: ${{ secrets.MACOS_CERT_PASSWORD }}
        run: |
          security create-keychain -p "action_keychain" build.keychain
          security default-keychain -s build.keychain
          security unlock-keychain -p "action_keychain" build.keychain
          security set-keychain-settings -lut 21600 build.keychain
          echo "$CERTIFICATE_P12" | base64 --decode > certificate.p12
          security import certificate.p12 -k build.keychain \
            -P "$CERTIFICATE_PASSWORD" -T /usr/bin/codesign
          security set-key-partition-list -S apple-tool:,apple:,codesign: \
            -s -k "action_keychain" build.keychain

      # 2. 生成 Xcode 工程
      - name: Generate Project
        run: xcodegen generate

      # 3. 构建 App
      - name: Build App
        run: |
          xcodebuild -scheme RClick \
            -configuration Release \
            -derivedDataPath build \
            -destination "generic/platform=macOS" \
            CODE_SIGNING_ALLOWED=NO \
            ARCHS=arm64 \
            ONLY_ACTIVE_ARCH=NO \
            build

          APP_PATH=$(find build -name "RClick.app" -path "*/Products/Release/*" | head -n 1)
          echo "APP_PATH=$APP_PATH" >> $GITHUB_ENV

      # 4. 分层签名
      - name: Code Sign App
        run: |
          CERT_NAME=$(security find-identity -v -s "Developer ID Application" | awk -F'"' '{print $2}')

          # 签名 Frameworks
          find "$APP_PATH/Contents/Frameworks" -type f -perm +111 \
            -exec codesign --force --options runtime --sign "$CERT_NAME" {} \;

          # 签名 Extensions
          find "$APP_PATH/Contents" \( -name "*.framework" -o -name "*.appex" \) \
            -exec codesign --force --options runtime --sign "$CERT_NAME" {} \;

          # 签名主 App
          codesign --force --options runtime \
            --entitlements "RClick.entitlements" \
            --sign "$CERT_NAME" "$APP_PATH"

      # 5. 打包并公证
      - name: Notarize
        env:
          APPLE_ID: ${{ secrets.APPLE_ID }}
          APPLE_PASSWORD: ${{ secrets.APPLE_APP_SPECIFIC_PASSWORD }}
          TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
          KEY_ID: ${{ secrets.NOTARY_KEY_ID }}
          ISSUER_ID: ${{ secrets.NOTARY_ISSUER_ID }}
          PRIVATE_KEY: ${{ secrets.NOTARY_PRIVATE_KEY }}
        run: |
          ditto -c -k --sequesterRsrc --keepParent "$APP_PATH" RClick.zip
          echo "$PRIVATE_KEY" > AuthKey_${KEY_ID}.p8

          RESULT=$(xcrun notarytool submit RClick.zip \
            --key-id "$KEY_ID" \
            --issuer "$ISSUER_ID" \
            --key "AuthKey_${KEY_ID}.p8" \
            --team-id "$TEAM_ID" \
            --wait --output-format json)

          STATUS=$(echo "$RESULT" | jq -r '.status')
          if [ "$STATUS" != "Accepted" ]; then
            echo "Notarization failed"
            exit 1
          fi

      # 6. Staple 公证票据
      - name: Staple Ticket
        run: |
          xcrun stapler staple "$APP_PATH"
          spctl --assess --type execute --verbose=4 "$APP_PATH"

      # 7. 创建 DMG
      - name: Create DMG
        run: |
          brew install create-dmg
          cp -R "$APP_PATH" ./RClick.app

          create-dmg \
            --volname "RClick" \
            --window-size 600 400 \
            --icon-size 100 \
            --icon "RClick.app" 150 200 \
            --hide-extension "RClick.app" \
            --app-drop-link 450 200 \
            "RClick-v${{ steps.version.outputs.version }}.dmg" \
            "./RClick.app"

      # 8. 创建 Release
      - name: Create Release
        uses: softprops/action-gh-release@v2
        with:
          files: |
            RClick-v${{ steps.version.outputs.version }}.dmg
          generate_release_notes: true

      # 9. 失败时清理 tag
      - name: Cleanup Tag on Failure
        if: failure()
        run: |
          git push origin --delete "${{ github.ref_name }}"
```

### 9.5 签名与公证流程

```
构建 → 签名 → 公证 → Staple → 分发
          ↓        ↓
     Apple 扫描   嵌入票据
```

**签名**：使用 `codesign` 和 Developer ID 证书，证明"我是谁"
**公证**：使用 `notarytool` 提交给 Apple 服务器扫描，证明"我是安全的"
**Staple**：将公证票据嵌入 App，离线用户也能运行

## 十、风险评估与缓解

### 10.1 破坏性变更
**风险**: 通信协议重构可能影响兼容性
**级别**: 中等
**缓解**:
- Enum 的 rawValue 保持与原有字符串相同
- 支持向后兼容的消息解析
- 逐步迁移，保留过渡期

### 9.2 消息同步
**风险**: 主程序和 Extension 消息不同步
**级别**: 中等
**缓解**:
- 使用心跳机制确认连接
- 主程序启动时主动发送配置
- Extension 定期请求配置刷新

### 9.3 Extension 体积
**风险**: 引入 SwiftData 增加 Extension 体积
**级别**: 低
**缓解**:
- 本方案中 Extension 不引入 SwiftData
- Extension 只使用内存缓存，体积极小

### 9.4 性能影响
**风险**: 消息传递可能比直接读取慢
**级别**: 低
**缓解**:
- 使用内存缓存，减少消息频率
- 只在配置变更时发送完整配置
- 实测性能，优化消息频率
