## 一、计划摘要

本计划描述了 RClick 架构重构的详细实施步骤，目标是实现**瘦 Extension 架构**，将 FinderSync 扩展简化为纯渲染层。

### 实施顺序调整（根据 Code Reviewer 建议）

**原顺序**：Phase 1 → 2 → 3 → 4 → 5 → 6 → 7 → 8

**调整后**：Phase 1 → **5** → **2** → 3 → 4 → 6 → 8 → 7

**调整理由**：
- Phase 5（完全磁盘访问权限）提前到 Phase 2 前，避免删除旧 Bookmark 逻辑后出现权限真空
- Phase 8（脚本功能）在 Phase 7（清理）前完成，便于使用新通信协议调试
- Phase 7 作为统一清理和验证放在最后

### 优先级说明

| 优先级 | 说明 |
|--------|------|
| P0 | 最高优先级，核心功能，必须完成 |
| P1 | 高优先级，重要优化 |
| P2 | 中优先级，改进体验 |

```swift
private func getUniqueTag(for rid: String) -> Int {
    var newTag = Int.random(in: 1 ... Int.max)
```

**修改后**:

```swift
    let tag = nextTag
    nextTag += 1
    tagRidDict[tag] = rid
    return tag
}
```

```swift
@MainActor
public class IconCache {
    public static let shared = IconCache()

    private var memoryCache: [String: NSImage] = [:]
    private let iconSize = CGSize(width: 32, height: 32)

    private init() {}

    public func icon(for url: URL) -> NSImage {
        let cacheKey = url.path
        if let cached = memoryCache[cacheKey] {
            return cached
        }

        let icon = NSWorkspace.shared.icon(forFile: url.path)
        icon.size = iconSize
        memoryCache[cacheKey] = icon
        return icon
    }

    public func preloadIcons(for urls: [URL]) {
        for url in urls {
            _ = icon(for: url)
        }
    }

    public func clearCache() {
        memoryCache.removeAll()
    }
}
```

```swift
public class PermissionChecker {
    /// 多点验证完全磁盘访问权限
    public static func hasFullDiskAccess() -> Bool {
        let testPaths = [
            "/Library/Application Support",
            "~/Library/Application Support",
            "/Library/Preferences"
        ]
        
        var accessibleCount = 0
        for path in testPaths {
            let expandedPath = NSString(string: path).expandingTildeInPath
            if FileManager.default.isReadableFile(atPath: expandedPath) {
                accessibleCount += 1
            }
        }
        return accessibleCount >= 2  // 至少 2 个路径可访问
    }
    
    /// 打开系统设置请求权限
    public static func openFullDiskAccessSettings() {
        NSWorkspace.shared.open(URL(string: "x-apple.systempreferences:com.apple.preference.security?Privacy_AllFiles")!)
    }
}
```

### Phase 6: Extension 重连机制 ⭐⭐ P1

**目标**：解决主 App 已打开但 Extension 未加载的问题

**预计时间**：2-3 小时

**风险**：低

#### 设计原则（必须遵守）

| 原则 | 说明 | 违反后果 |
|------|------|----------|
| **Notification ≠ 状态** | 通知只能当"信号"，不能当"真相" | 丢消息=状态错乱 |
| **必须存在状态快照** | 快照需持久化，否则无法恢复一致性 | 重启后数据丢失 |
| **Extension 必须支持主动拉取** | 不能只靠推送 | 无法自愈 |
| **所有通信必须幂等** | 重复消息不能出错 | 重复执行风险 |
| **版本号必须单调递增** | 版本号需持久化，防止回滚 | 旧配置覆盖新配置 |

#### 步骤 6.1: 状态快照机制（P0）⭐ 核心

**文件**：`RClick/RClickApp.swift`

**修改内容**：
- 添加 `lastMenuSnapshot: Data?` 和 `menuVersion: Int`
- **新增**：添加 `@AppStorage` 持久化存储
- 每次菜单配置变更时更新快照和版本号
- **新增**：App 启动时从持久化恢复快照
- 注册 `requestConfig` 消息处理器

**核心设计**：

```swift
// 持久化快照（真相源）
@AppStorage("lastMenuSnapshot") private var savedSnapshot: Data?
@AppStorage("menuVersion") private var savedMenuVersion: Int = 0

// 内存中的快照（运行时使用）
private var lastMenuSnapshot: Data?
private var menuVersion: Int = 0

// App 启动时恢复快照
func restoreSnapshot() {
    if let data = savedSnapshot {
        lastMenuSnapshot = data
        menuVersion = savedMenuVersion
        logger.info("Restored snapshot from persistent storage (version=\(menuVersion))")
    }
}

// 状态变更时
func updateMenuConfig() {
    menuVersion += 1
    let snapshot = MenuSnapshot(version: menuVersion, config: buildMenuConfig())
    lastMenuSnapshot = try? JSONEncoder().encode(snapshot)
    
    // 持久化保存
    savedSnapshot = lastMenuSnapshot
    savedMenuVersion = menuVersion
    
    // 发送 running 消息时携带 snapshot
    sendRunningMessage(snapshot)
}

// 响应 Extension 的 config 请求
func handleConfigRequest() {
    if let data = lastMenuSnapshot {
        Messager.shared.sendToExtension(.menuConfig, data: data)
    }
}
```

**running 消息携带 MenuSnapshot**：

```swift
func sendRunningMessage(_ snapshot: MenuSnapshot) {
    Messager.shared.sendToExtension(.running, data: snapshot)
}
```

#### 步骤 6.2: 主动请求机制（P0）⭐ 核心

**文件**：`FinderSyncExt/FinderSyncExt.swift`

**修改内容**：
- 添加 `sendMenuRequest()` 方法
- `init()` 中启动时发送请求
- `menu(for:)` 中缓存为空时触发请求
- 接收更新时检查版本号（防重复、防乱序）
- **新增**：版本号检查支持时间戳辅助校验

```swift
class FinderSyncExt: FIFinderSync {
    private var currentVersion: Int = 0
    private var currentTimestamp: TimeInterval = 0
    private var cachedMenuConfig: MenuConfigPayload?

    override init() {
        super.init()
        setupMessageHandlers()
        // 启动时主动请求配置
        sendMenuRequest()
    }

    private func handleRunning(_ data: Data?) {
        guard let snapshot = decode<MenuSnapshot>(data) else { return }
        // 版本检查：只接受更新的版本
        if snapshot.version > currentVersion ||
           (snapshot.version == currentVersion && snapshot.timestamp > currentTimestamp) {
            currentVersion = snapshot.version
            currentTimestamp = snapshot.timestamp
            cachedMenuConfig = snapshot.config
        }
    }

    private func handleMenuConfig(_ data: Data?) {
        guard let config = decode<MenuConfigPayload>(data) else { return }
        cachedMenuConfig = config
    }

    // 菜单构建：只读缓存
    override func menu(for menuKind: FIMenuKind) -> NSMenu {
        guard let config = cachedMenuConfig else {
            // 缓存为空，触发请求
            sendMenuRequest()
            return NSMenu(title: "RClick (loading...)")
        }
        // 使用缓存构建菜单...
    }
}
```

#### 步骤 6.3: 心跳机制（P1）

**文件**：`RClick/RClickApp.swift` 和 `FinderSyncExt/FinderSyncExt.swift`

**修改内容**：
- **Main App 端**：
  - 在 `applicationDidFinishLaunching` 中发送 `running` 消息（携带 `MenuSnapshot`）
  - 启动重试定时器：每 5 秒发送一次，持续 30 秒（6 次）
  - 注册 `heartbeat` 消息处理器，重置超时定时器（15 秒）
  
- **Extension 端**：
  - 启动定时器：每 10 秒发送 `heartbeat` 给主程序

```swift
// Main App 端
private var heartbeatTimeoutTimer: Timer?
private var isExtensionOnline: Bool = false

func handleHeartbeat() {
    isExtensionOnline = true
    // 重置超时定时器
    heartbeatTimeoutTimer?.invalidate()
    heartbeatTimeoutTimer = Timer.scheduledTimer(withTimeInterval: 15, repeats: false) { [weak self] _ in
        self?.handleHeartbeatTimeout()
    }
}

// Extension 端
Timer.scheduledTimer(withTimeInterval: 10, repeats: true) { _ in
    Messager.shared.sendToMain(.heartbeat)
}
```

```swift
    sendRunningMessageWithRetry()
```

**心跳方向说明**：

| 方向 | 消息类型 | 频率 | 目的 |
|------|----------|------|------|
| Main → Ext | `running` | 启动时 + 重试 3-4 次（指数退避：2s→4s→8s） | 告知 Extension 主程序已启动 |
| Ext → Main | `heartbeat` | 每 10 秒 | 告知主程序 Extension 存活 |

**参数调整说明**：
- **重试次数**：从 6 次减少到 3-4 次，减少用户等待时间
- **重试间隔**：采用指数退避（2s, 4s, 8s），总时长从 30 秒降至 14 秒
- **超时阈值**：从 15 秒调整为 25-30 秒（心跳间隔的 2.5-3 倍），避免误判

```swift
func detectDangerousCommands(_ scriptContent: String) -> [String] {
    let dangerousPatterns = [
        "rm -rf /", "rm -rf ~", "sudo",
        "dd if=/dev/zero", ":(){ :|:& };:",
        "> /dev/disk", "mkfs", "newfs"
    ]
    return dangerousPatterns.filter { scriptContent.contains($0) }
}
```

**2. 执行前用户确认**：
```swift
func confirmScriptExecution(script: RCScript, files: [String]) -> Bool {
    let alert = NSAlert()
    alert.messageText = "确认执行脚本？"
    alert.informativeText = """
        脚本名称：\(script.name)
        脚本内容预览：\(script.scriptContent.prefix(200))...
        将处理 \(files.count) 个文件
        """
    return alert.runModal() == .alertFirstButtonReturn
}
```

**3. 执行超时控制**：
```swift
func executeScriptWithTimeout(_ script: String, files: [String], timeout: TimeInterval = 30) {
    let process = Process()
    process.executableURL = URL(fileURLWithPath: "/bin/bash")
    process.arguments = ["-c", script, "--"] + files
    try? process.run()
    
    DispatchQueue.global().asyncAfter(deadline: .now() + timeout) {
        if process.isRunning { process.terminate() }
    }
    process.waitUntilExit()
}
```
